# v0.25.0

## Added

- Added bounded claim and proposal triage artifacts that record problem truth,
  implementation duplication, spec and active-change conflicts, readiness,
  disposition, and evidence bindings.
- Added deterministic triage validation and repository evidence collection for
  ordinary GitHub Issues and local proposal review.

## Changed

- Prepared and ordinary Issues now acquire a minimal claim lock before triage;
  executable work proceeds through the existing claim state machine only after
  the evidence-bound triage result passes.
- Complex Issues remain claimed until independently executable child Issues are
  linked and all are open `type:change` items with a unique `status:ready`.
- Triage dispositions map to the existing Buddy statuses without introducing a
  second label system. Terminal dispositions release the claim while pending
  triage and unfinished series decomposition retain it.

## Fixed

- Preserved stale-claim recovery for expired or foreign claimed Issues instead
  of making recovery unreachable behind the active-claim triage verifier.
- Made series-parent type and tracking transitions transactional, including
  rollback and remote post-mutation verification.
- Restored `status:ready` after ordinary claim failures only when Claim Release
  succeeds, while preserving verified terminal triage states during release
  retries.
- Transferred cleanup ownership to resumed claims so downstream dependency,
  coupling, and Development failures cannot leave an active claim behind.

# v0.16.0

## Changes

- Added target-aware Buddy Auto context binding for user-specified work:
  `OPENSPEC_BUDDY_AUTO_TARGET_ISSUE` and `OPENSPEC_BUDDY_AUTO_TARGET_PR`.
- Prevented a target issue from being overwritten by an ambient current PR from
  a reused worktree.
- Changed target PR inference to read the specified PR for origin issue and
  head SHA instead of reading the current branch PR.
- Added an issue-only target claim stage to the auto driver. The driver now
  runs `claim-issue.sh <issue>` and records a signed `claimed` receipt before
  handing off implementation.
- Updated Buddy Auto driver state documentation and main skill guidance for
  target issue and target PR entry.
- Added regression coverage for target issue, target PR, no-context safety, and
  current-head receipt behavior.

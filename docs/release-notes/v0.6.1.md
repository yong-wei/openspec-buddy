# v0.6.1

## Highlights

- Hardened Buddy pre-archive flow so active change/delta specs are validated
  before `openspec archive` moves the change into the archive tree.

## Changes

- Updated `openspec-buddy apply` and `openspec-buddy-auto` to require
  `openspec validate <change_id> --strict` before pre-archive.
- Updated the Buddy Auto execution-loop reference to preserve the full
  validation order: completed tasks, active change validation, archive, then
  affected main spec validation.
- Added eval coverage that fails if the active change validation gate is
  missing or placed after archive.
- Fixed the review-request helper gate so unresolved review threads, requested
  changes, or `P0/P1/P2` findings block duplicate review requests even when an
  older review also targets a stale head.

# v0.14.0

## Changes

- Simplified `openspec-buddy` and `openspec-buddy-auto` `SKILL.md` files into
  driver-first entrypoint documents with `<EXTREMELY_IMPORTANT>` gate blocks.
- Added `buddy-driver.mjs` for core Buddy phases. It reports repository state
  and the next legal helper command for claim, propose, apply, and achieve
  flows.
- Added `buddy-auto-driver.mjs` for Buddy Auto phases. It reports the next
  legal auto action, writes signed local receipts after successful stage helper
  execution, and refuses unsigned or tampered receipts.
- Added `core-lifecycle.md` and `driver-states.md` so detailed phase rules live
  in references instead of the main skill files.
- Hardened the auto local-only path so `--no-pr` is accepted only for explicit
  local-only `--change` runs, not issue-backed work.
- Added regression coverage for core `--run-next` ordering, auto receipt
  progression, forged receipt rejection, and invalid `--no-pr` combinations.

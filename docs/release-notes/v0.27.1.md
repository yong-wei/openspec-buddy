# v0.27.1

## 修复

- Buddy Auto lite 无目标选择不再一次性读取全部开放、关闭 Issue 与 Pull Request；选择器改为只查询开放的 Buddy 活跃状态，并将实际 Issue 候选集合限制为 50 个。
- GitHub REST `/issues` 返回的 Pull Request 会在候选计数前排除，避免少量 Issue 与同标签 PR 混合时被误判为超过上限。
- 显式 `--issue` 直接读取目标 Issue；`--change` 和重复映射核验按每页 50 条处理历史记录，不再通过 `--paginate --slurp` 聚合全部输出。

## 资源保护

- Issue 列表响应仅保留选择和映射校验所需字段。
- 子进程输出缓冲区上限调整为 16 MiB，覆盖单页 Issue 正文及评论读取，同时保持每次列表请求有界。
- 实际开放 Buddy Issue 超过 50 个时明确停止，不在不完整候选集上认领。

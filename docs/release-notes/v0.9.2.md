# v0.9.2

## Highlights

- Hardened Buddy Project status synchronization when GitHub only returns
  project titles in issue or PR metadata.
- Improved issue relationship verification so cross-series dependencies are
  included in the checked set without trusting stale relationship cache data.

## Changes

- `set-project-status.sh` and `set-project-date.sh` now recover editable
  Project item ids through target-scoped GraphQL when REST metadata only
  exposes the Project title.
- Project item recovery now prefers exact Project id matches and fails closed
  when only same-title items from a different Project are available.
- `verify-issue-relationships.sh` expands parent, child, blocking, and
  blocked-by endpoints into the verification set, including cross-series
  dependencies.
- Relationship cache refresh now invalidates stale relationship and issue cache
  entries when GraphQL no longer returns a requested issue.
- PR label inheritance now accepts both REST label arrays and GraphQL
  `labels.nodes` payloads from cached issue data.

# v0.23.0

## Added

- Added a deterministic `.buddy/proposal-review.yaml` contract for proposal
  shape, vertical-slice validity, dependency completeness, wide-refactor
  strategy, and series children.
- Added proposal-shape validation to the manual Buddy `propose` path after
  issue-body validation and before any GitHub Issue mutation.

## Changed

- Defined executable child changes as independently claimable, testable,
  reviewable, and deliverable through one PR; implementation tasks inside a
  change may still span database, API, UI, and tests.
- Defined GitHub native `blockedBy` relationships as dependency truth, with
  Buddy metadata retained only as a mirror.
- Defined `expand-migrate-contract` as the supported strategy for broad
  mechanical migrations that cannot form independently valid pseudo-slices.

## Fixed

- Rejected contradictory single-change manifests with non-empty children and
  hardened unknown-field validation against special object keys.
- Accepted both verb and noun forms for the expansion, migration, and
  contraction phases described by the documentation.
- Allowed Buddy Auto single mode to recover an externally merged PR only after
  independently verifying the merge truth and binding signed recovery evidence
  to the current issue, change, PR, and head.

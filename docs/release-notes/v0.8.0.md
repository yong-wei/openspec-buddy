# v0.8.0

## Highlights

- Reduced routine GitHub GraphQL usage across Buddy selection, relationship
  verification, and review waiting by moving high-frequency paths to REST-first
  fetching with single-run cache reuse.

## Changes

- Added `skills/openspec-buddy/scripts/github-fetch.sh` as a shared GitHub
  access layer for repository resolution, REST bundles, issue-body fallback,
  batched relationship GraphQL queries, and review-thread GraphQL fetches.
- Updated `list-ready-change-relationships.sh` to fetch open issues through
  REST first, prefilter candidates locally, normalize spaced status labels such
  as `status: ready`, and only query GraphQL relationships for remaining
  candidates.
- Updated `verify-issue-relationships.sh`, `verify-review-clear.sh`, and
  `wait-for-review-clear.sh` so review waiting reuses REST payloads within a
  single run, refreshes commit and thread state when the PR head changes, and
  limits GraphQL to review-thread truth checks.
- Tightened local-only fallback semantics in `select-next-change.sh` so mixed
  local-only and issue-backed runs warn explicitly when GitHub relationship
  lookups fail instead of silently presenting a pure local-only result.
- Updated `link-issue-parent.sh` and `close-completed-series-parent.sh` to
  reuse the shared repo-resolution layer while remaining compatible with the
  current `gh api graphql` CLI surface.
- Added eval coverage for batched relationship fetches, body fallback parsing,
  mixed local-only fallback warnings, review-thread cache refresh, wait-helper
  commit-cache refresh, verifier timeout handling, and GitHub CLI GraphQL
  compatibility.

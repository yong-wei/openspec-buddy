# v0.19.0

## Added

- Added an opt-in Buddy Auto multi-lane scheduler for goal mode. The lane
  driver can park a clean, pushed PR while it waits for online Codex review,
  claim another executable issue, and later resume whichever waiting lane gets
  review feedback first.
- Added lane state and lane switch guards so multi-lane mode records issue,
  change id, branch, PR, head SHA, review request timing, retry count, and
  blocked state before switching foreground work.
- Added lightweight review probing helpers:
  `probe-review-state.sh` and `check-review-clear-once.sh`. The existing
  blocking wait helper remains available for single-lane mode.
- Added selector exclusions for active lane issues so goal mode does not
  re-claim work already parked in the current multi-lane state.
- Added a single Buddy Auto controller entrypoint, `buddy-auto.mjs`, that
  persists single-lane or multi-lane mode, target seeds, active interrupts, and
  review-fix state before delegating to internal drivers.

## Fixed

- Hardened multi-lane state transitions for review-yield, review-fix,
  merge-ready, local-only selection, blocked lanes, and no-candidate goal-loop
  completion.
- Ensured a fully completed lane does not prevent `DONE no-available-changes`,
  while blocked-only lane state reports `BLOCKED blocked-lanes` instead of
  silently completing.
- Ensured blocked lanes do not stop active waiting lanes from being probed.
- Converted Buddy Auto `HANDOFF` and `BLOCKED` into persistent controller
  interrupts so agents resume by running the controller again instead of
  choosing helper scripts.
- Preserved review-fix follow-up across process restarts by storing
  `reviewFix.pending`, preventing the response gate from being skipped when an
  environment variable is lost.
- Removed deterministic helper commands from the agent-facing Buddy Auto skill
  path; single-lane and multi-lane drivers remain compatibility internals.

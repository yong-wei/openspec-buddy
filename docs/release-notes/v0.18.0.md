# v0.18.0

## Changed

- Promoted Buddy Auto driver from a stage hint helper to a deterministic state
  machine executor. It now continues through claim, exact PR discovery,
  review coordination, review wait, merge gates, and achievement truth checks
  until it reaches a real `HANDOFF`, `BLOCKED`, or terminal `DONE`.
- Added exact issue-to-PR bridging with `find-issue-pr.sh`, so a target issue
  automatically enters the PR state machine only when PR evidence matches the
  issue. Ambient current PRs remain ignored.
- Added post-merge achievement support with `verify-achieved-truth.mjs` and
  `mark-achieved-post-merge.sh`, allowing issue and Project terminal state to
  be repaired from the bound coordination branch after merge.
- Added `reply-review-thread.sh` and `review-response-gate.sh --reply-plan` so
  review-fix loops can reply in the same thread, resolve addressed threads,
  and verify final thread state without hand-written GraphQL.
- Reduced review request and wait overhead: first review requests no longer
  query reviewThreads by default, wait startup uses lightweight thread status,
  and idle wait polling uses narrow PR signatures.
- Standardized `--help` handling for high-frequency Buddy shell helpers.

## Fixed

- Fixed GraphQL thread helper failure propagation so callers cannot treat a
  failed reviewThreads fetch as a successful cache file.
- Split review-response-gate resolve success from final transient verification
  failure and report `safe_to_rerun: true` when rerunning is safe.

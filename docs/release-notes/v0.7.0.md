# v0.7.0

## Highlights

- Added explicit local-only coordination paths for `openspec-buddy propose --no-issue`
  and `openspec-buddy-auto --no-pr`, while preserving the existing one-change,
  one-issue, one-PR GitHub-backed workflow for issue-coordinated changes.

## Changes

- Documented `openspec-buddy propose --no-issue` as a local-only exception that
  creates the OpenSpec change without creating or updating a GitHub Issue.
- Documented and constrained `openspec-buddy-auto --no-pr` so it only applies
  to local-only `--no-issue` changes; issue-backed changes still require the
  normal PR, review, and issue/project synchronization flow.
- Updated `select-next-change.mjs` to preserve structured active change data,
  recognize local-only markers, reject ambiguous `issue: null`, and avoid
  falling back to local-only mode when a malformed issue already claims the same
  `change_id`.
- Updated `select-next-change.sh`, `load-config.sh`, and `check-config.sh` to
  support a minimal local-only configuration path and to fall back locally when
  local-only changes exist but GitHub relationship queries are unavailable.
- Added eval coverage for local-only selector behavior, local-only config
  requirements, and the narrowed `--no-pr` semantics.

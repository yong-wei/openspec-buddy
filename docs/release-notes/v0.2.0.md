# v0.2.0

## Highlights

- Added `openspec-buddy claim` as the intake path for ordinary open GitHub issues.
- Updated `openspec-buddy-auto` to begin each iteration from claimable issues before applying prepared OpenSpec changes.

## Changes

- Documented the one-issue intake flow: claim first, then classify the issue as a simple executable change or a parent issue that must be decomposed into child changes.
- Added hidden OpenSpec Buddy metadata support for adopted open issues while keeping existing front matter metadata parsing.
- Added scripts for selecting the smallest claimable open issue, building Buddy metadata, and claiming ordinary issues through GitHub Development branch locks.
- Updated selection and status references so `status:claimed` issues owned by the current viewer can continue into `apply`.
- Updated the README workflow description to explain how open issue claiming works with OpenSpec proposal and implementation skills.

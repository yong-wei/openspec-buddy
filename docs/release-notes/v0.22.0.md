# v0.22.0

## Added

- Added live claim-truth verification so local receipts and lane state can no
  longer authorize claim-sensitive work without confirming GitHub state.
- Added cache hit, miss, request, and stale-recovery metrics for GitHub-backed
  coordination reads.
- Added an explicit `OPENSPEC_BUDDY_ALLOW_NO_CI` project policy for repositories
  that intentionally do not run CI on pull requests.

## Changed

- Demoted persisted controller, lane, and receipt state to recovery and audit
  evidence rather than remote coordination truth.
- Made claim, review, merge, archive, and Project mutations refresh the relevant
  remote state before proceeding.
- Hardened merge CI discovery with a fixed observation window covering check
  suites, check runs, and legacy statuses; incomplete API pages now fail closed.

## Fixed

- Prevented stale local claim state from skipping a required live claim after a
  release, restart, worktree switch, or timeout recovery.
- Prevented repositories without applicable PR CI from remaining permanently
  blocked by GitHub's empty combined-status `pending` response.
- Prevented optional CI startup races, late failures, and truncated check lists
  from being mistaken for a successful merge gate.

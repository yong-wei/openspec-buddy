# v0.9.1

## Highlights

- Stabilized the fine-grained Buddy cache/signal rollout after the initial
  `v0.9.0` release by fixing review-request cache invalidation, hardening cache
  writes, and aligning the packaged test entrypoint with the published tarball.

## Changes

- Invalidated cached PR REST bundles after `request-pr-review.sh` posts or
  refreshes a review request so same-run review verification can observe the
  new top-level comment immediately.
- Switched cache entry writes and GraphQL-derived issue/relationship cache
  writes to atomic temp-file replacement to reduce partial-write risk when
  multiple Buddy processes share a cache directory.
- Included `test/` in the npm publish whitelist and routed `npm test` through
  `test/run-all-tests.mjs` so the published package retains its release
  verification entrypoint.
- Synced skill docs and release notes with the shipped cache semantics,
  including `OPENSPEC_BUDDY_CACHE_SIGNAL_REMOTE` behavior and relationship
  cache hydration after node-only status changes.

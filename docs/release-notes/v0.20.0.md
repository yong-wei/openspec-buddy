# v0.20.0

## Fixed

- Classified a missing response after the latest current-head Codex review
  request as waitable instead of a hard review-check failure.
- Made controller-owned merge authorization transactional: failed merge helpers,
  non-merged results, and mismatched merge evidence now clear the matching
  authorization receipt.
- Made foreground review waits stop immediately with the unavailable status when
  Codex quota or service capacity is unavailable, preserving the evidence and
  avoiding an automatic retry request.
- Hardened Checks-only merge handling and bound the merge command to the
  repository that passed the preceding remote truth checks.
- Preserved the current-head review request state when a previously unavailable
  lane recovers.

## Verification

- `npm test`
- `npm pack --dry-run`
- Focused regression tests for review wait classification, foreground
  unavailable handling, current-head request recovery, and merge authorization
  cleanup.

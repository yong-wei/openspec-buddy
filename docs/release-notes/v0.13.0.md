# v0.13.0

## Changes

- Added `validate-issue-body.mjs` for `openspec-buddy propose` issue bodies.
  The validator first reuses existing Buddy metadata validation, then enforces
  the Buddy collaboration contract:
  `## Acceptance Checklist`, unchecked `AC-n` items, AC evidence, `## Tasks`,
  task `Covers: AC-n`, task `Acceptance`, task `Evidence`, and task
  `Reviewer Check`.
- The validator also rejects duplicate or non-sequential AC ids and fails when
  any AC item is not covered by at least one task.
- Updated `openspec-buddy propose` guidance so GitHub Issue creation must pass
  the new issue-body validator, not just `parse-issue-metadata.mjs`.
- Preserved compatibility for legacy claimed issues: existing claim/apply paths
  still use metadata parsing and do not require old collaborator issues to have
  AC ids.
- Added regression coverage for missing checklist sections, missing task-to-AC
  binding, unknown AC references, orphan AC items, duplicate/non-sequential AC
  ids, missing task evidence, and checked AC items during propose.

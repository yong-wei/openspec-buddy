# v0.5.0

## Highlights

- PR review clearance now understands the current-head review cycle instead of
  treating any broad PR comment as enough to merge.
- Local npm-token publishing is now documented through a committed
  `.env.release-package.example` template while keeping `.agents/` ignored.

## Changes

- `request-pr-review.sh` refreshes the configured review request after a new PR
  head commit instead of reusing a stale request.
- `verify-review-clear.sh` now builds PR review state from REST and GraphQL
  data, including review requests, issue comments, commits, review comments,
  and review threads.
- `verify-review-clear.mjs` allows a top-level clear comment only when it
  follows a review request posted after the current head commit, and prints the
  matched comment excerpt, time, and URL as the judgment record.
- `openspec-buddy-auto` review guidance now requires agents to use the helper
  output instead of manually inferring clearance from broad comment text.
- Added `.env.release-package.example` for local `npm publish` token setup
  without committing the real `.env.release-package` or `.agents/`.

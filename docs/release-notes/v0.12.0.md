# v0.12.0

## Changes

- Changed `wait-for-review-clear.sh` to use two-stage review waiting:
  lightweight PR REST probes during idle polling, then full PR REST bundle and
  `verify-review-clear.sh` only after the lightweight PR state changes.
- Lowered the default review poll interval from 120 seconds to 60 seconds after
  the initial 300 second wait.
- Added a first-timeout retry path: after one 900 second wait window without a
  current-head clean review, the wait helper calls `request-pr-review.sh
  --force` with generated retry context and waits one more 900 second window.
- Added `request-pr-review.sh --force` so retry requests can append context
  after the fixed configured review request even when a current-head request is
  already present.
- Allowed `verify-current-head-review-request.sh` to reuse a freshly refreshed
  PR REST bundle when called from the wait helper.
- Updated Buddy Auto review-waiting guidance to require human intervention
  after the second wait window times out.

# v0.15.0

## Changes

- Changed both Buddy drivers to default execution mode. Normal operation now
  runs `buddy-driver.mjs` or `buddy-auto-driver.mjs` without arguments.
- Changed driver success output to a compact `DONE` result and helper failure
  output to `BLOCKED` with diagnostics. Successful helper stdout/stderr is
  captured instead of streamed.
- Added `HANDOFF` output for phases that require agent-owned work rather than
  another deterministic helper.
- Let Buddy Auto continue deterministically from `mark-review` into
  `wait-for-review-clear.sh`, so review waiting is entered through one driver
  invocation instead of manual sleeps or polling.
- Added a signed `merge_gates_passed` receipt so repeated driver runs after
  review clearance hand off to merge/achievement instead of repeatedly running
  the same final review verification.
- Added environment context support for Buddy Auto
  (`OPENSPEC_BUDDY_AUTO_ISSUE`, `OPENSPEC_BUDDY_AUTO_PR`,
  `OPENSPEC_BUDDY_AUTO_CHANGE`) and kept old CLI options as diagnostic
  compatibility controls.
- Updated both skill entrypoints to require silent waiting after driver start:
  no commands, no GitHub queries, no time checks, and no progress output until
  the driver returns `DONE`, `BLOCKED`, or `HANDOFF`.

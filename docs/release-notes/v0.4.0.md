# v0.4.0

## Highlights

- `openspec-buddy propose` now creates the local OpenSpec change by default via
  `openspec-propose` and registers the matching GitHub Issue as one workflow.
- Codex review thread resolution now goes through a Buddy helper that verifies
  the GitHub thread is actually resolved before the merge loop continues.

## Changes

- Clarified that proposed issues must include the full applicable coordination
  state by default: labels, parent/dependency links, Project membership, and
  Project `Todo` status.
- Updated the user README so Buddy propose is no longer described as only a
  GitHub Issue registration step.
- Added a regression eval covering the default local-artifact behavior for
  propose mode.
- Added `resolve-review-thread.sh` and an eval fixture so automation cannot
  silently resolve review threads with an unchecked raw GraphQL mutation.

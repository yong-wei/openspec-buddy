# v0.10.0

## Features

- Added issue Acceptance Checklist guidance with task-to-AC binding, evidence, and independent reviewer approval before AC checkoff.
- Added a bound worktree guard for permanent coordination worktrees through `buddy.boundBranch`, `buddy.boundBase`, and `buddy.worktreeAlias`.
- Made `sync-base-branch.sh` fail fast on detached or wrong-branch entry when a bound worktree is configured, and fast-forward the bound branch to `buddy.boundBase`.
- Blocked claim entry before any GitHub query when a bound worktree is not on its configured coordination branch.
- Required active claim `coordination_branch` to match the configured bound branch.

## Fixes

- Preserved full Codex code review scope while adding extra checks for Acceptance Checklist coverage, scope drift, and unregistered requirements.
- Clarified that issue AC-linked task checkoff is independent from OpenSpec `tasks.md` completion.

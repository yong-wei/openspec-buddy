# v0.26.0

## 不兼容变化

- `scripts/buddy-auto.mjs` 无参数调用现在进入 lite。原有 controller 工作流改为显式 `scripts/buddy-auto.mjs full`。
- 不提供兼容环境变量、旧配置自动探测或弃用期。既有 full 状态文件保持原位置与格式，显式 full 会继续读取，不需要迁移。

## 新增

- 新增无持久状态的 lite 公开入口，支持无目标选择、`--issue <number>`、`--change <change_id>` 与受限的 Local-only `--no-pr`。
- 新增三个 lite 协调脚本，分别处理 Available Issue 选择、Git Refs 原子 Claim 与唯一 `status:*` 标签更新。
- 新增面向 GPT-5.6 主模型的 PR 合同：固定 Review Request、300/60/900 等待、一次 Timeout Retry、服务不可用立即停止、feedback 必复审和 latest-head Clearance 门禁。

## 变更

- Lite 将实现、测试、同步后的主 specs 与 OpenSpec archive 置于同一 PR；显式 Direct Integration Delivery 也保持同一交付单元。
- Local-only 默认创建 PR；只有完全没有映射 Issue 且用户明确指定 `--no-pr` 时，才允许 fast-forward 直接集成。
- `openspec-buddy init` 默认只要求 `OPENSPEC_BUDDY_BASE_BRANCH`；manual Buddy 与 Auto full 使用 `openspec-buddy init --full` 保持原有完整配置。
- Auto references 现在明确标为 full-only，所有 controller 与恢复命令统一从公开 `buddy-auto.mjs full` 入口进入。

## 旧命令与新命令

```text
旧：scripts/buddy-auto.mjs
新：scripts/buddy-auto.mjs full

新默认 lite：scripts/buddy-auto.mjs
```

## 验证边界

- 完整执行了 lite entry、selector、Claim、status、skill contract 与 CLI init 测试。
- Full 只验证公开入口、参数透传、既有状态读取和 `DONE/HANDOFF/BLOCKED` 协议冒烟；本次没有运行 controller、cache、lane 完整回归，也没有运行 `npm test`。
- 另执行语法检查、`npm pack --dry-run` 与 `git diff --check`。

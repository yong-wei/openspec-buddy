# v0.9.3

## Highlights

- Hardened series parent closeout so parent tracking issues cannot remain open
  silently after all child changes reach terminal state.
- Added reconciliation for completed series parents after `mark-achieved.sh`.

## Changes

- `close-completed-series-parent.sh` now requires every child issue to be
  `CLOSED`, labeled `status:archived`, set to Project `Done`, and have Project
  `End` set before finalizing the parent issue.
- Closed child issues with Project `Done` and `End` but missing
  `status:archived` are reported as repairable terminal drift and cause the
  closeout script to fail.
- Project terminal checks now match the configured Project by id and title to
  avoid same-title Project ambiguity.
- Added `reconcile-completed-series-parents.sh` to scan open tracking series
  parents and finalize those whose child issues are complete.
- `mark-achieved.sh` now runs series parent reconciliation after syncing the
  completed child issue.
- Updated Buddy Auto and relationship references to document the stricter
  parent closeout postcondition.
- Recorded the repository workflow constraint that future changes should be
  developed in isolated worktrees, reviewed by subagents, merged back to main,
  and released from the cleaned main branch.

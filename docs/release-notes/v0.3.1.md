# v0.3.1

## Highlights

- Made claim/base alignment worktree-friendly: claim helpers no longer force a switch to the Buddy base branch when the current worktree is already aligned with `origin/$OPENSPEC_BUDDY_BASE_BRANCH`.
- Added a hard review-clearance gate so Codex review findings in review bodies, review comments, or unresolved review threads cannot be missed before merge.

## Fixes

- `sync-base-branch.sh` now fast-forwards only when the current branch is the configured Buddy base branch. In topic/worktree branches, it verifies that current `HEAD` matches `origin/$OPENSPEC_BUDDY_BASE_BRANCH` and leaves the branch unchanged.
- Added `verify-review-clear.sh` / `.mjs` and eval coverage for the failure mode where `gh pr view --comments` is empty but a `COMMENTED` Codex review still contains `P1` or `P2` findings.
- Updated `openspec-buddy-auto` merge gates so `COMMENTED` is not treated as a pass. `P0`, `P1`, and `P2` findings block merge until they are verified, addressed or justified with evidence, and followed by a clean review state.

# v0.3.0

## Highlights

- Hardened the Buddy PR review handoff so review requests, PR metadata, labels, assignees, Project state, and origin issue references are verified through shared helpers.
- Added a reusable base-branch synchronization helper and wired claim/apply/auto guidance through it.

## Changes

- Added `request-pr-review.sh` and made `mark-review.sh` post the configured `OPENSPEC_BUDDY_PR_REVIEW_REQUEST` before moving an issue to `status:in-review`.
- Added `verify-pr-coordination.sh` / `.mjs` to check PR base, draft state, required labels, mirrored assignees, Project status, origin issue traceability, Development-link policy, review-request comment, and OpenSpec change path scope.
- Added `build-pr-labels.mjs` so Buddy PR labels include `pr:*` labels plus inherited non-status coordination labels from the source issue.
- Added `sync-base-branch.sh` and updated claim/apply/auto documentation to rely on the shared base synchronization path.
- Updated the CLI prompt, README, and example env file so PR review requests are project configuration rather than auto-mode-only wording.
- Expanded eval coverage for PR labels, review request idempotency, base branch sync, and PR coordination verification.

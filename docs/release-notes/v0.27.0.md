# v0.27.0

## 新增

- Lite Review Window 新增一次性只读 `read-review-evidence.mjs` 快照入口，同时读取当前 PR head、顶层评论及 reactions、Pull Reviews、inline review comments 与 GraphQL review threads。
- 快照为各来源保留来源名称、作者、时间、URL 和原始内容，并按来源提供适用的绑定 commit、reaction 与线程状态，主模型不再自行拼接单渠道查询。
- 快照支持 PR 编号、当前仓库 PR 引用和完整 GitHub PR URL；完整 URL 使用其自身 owner、repository 与 PR number。

## 变更

- 默认 manual `propose` 回归轻量协调：保留已推送的本地 OpenSpec change、唯一 `change_id` Issue 映射、`type:change`、`status:ready` 与原生 `blockedBy`。
- triage、proposal-review、Testing Strategy、task-to-AC、Project 和完整标签矩阵不再作为默认 propose 硬闸门；历史完整 metadata 与 claim 阶段 triage 保持兼容。
- Lite Review 等待仍由主模型自主执行。Evidence 脚本不分类 clearance、不等待、不重试、不请求 Review、不缓存，也不写入远端状态。

## 正确性保护

- head commit 时间按 PR head SHA 直接读取，不依赖最多 250 项的 PR commits 列表。
- 所有 Review 来源读取完成后再次读取 PR；若 head SHA 在快照期间变化，整次读取失败，不返回跨 revision 的混合证据。
- REST 来源完整分页；GraphQL threads 或 thread comments 超过轻量读取上限时明确 fail closed，不把部分结果交给模型。

## 验证

- `npm test`
- `npm pack --dry-run`
- `git diff --check`
- 使用真实 PR 验证数字与完整 URL 快照
- 独立 Sol medium 审核与 current-head Codex Review 清场

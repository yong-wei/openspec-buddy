# v0.9.4

## Fixes

- Hardened Buddy claim coordination for multi-worktree and multi-agent races.
- Added a shared claim lock layer that reads GitHub truth before claim, writes only the minimal claim lock, and verifies the latest active claim through REST before any Development or Project updates.
- Prevented failed claim cleanup from deleting another agent's claim branch by requiring active-claim ownership before branch deletion.
- Added Claim Release comments so losing or interrupted claim attempts can release their own active claim without overwriting the winning agent.
- Defaulted selectors to skip `status:claimed` and expose stale-claim candidates only when no normal claimable issue is available.

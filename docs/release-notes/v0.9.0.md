# v0.9.0

## Highlights

- Added repository-local Buddy cache infrastructure and target-scoped fetches so
  routine status, review, and project workflows no longer depend on full
  project scans or unnecessary GraphQL reads.

## Changes

- Added `skills/openspec-buddy/scripts/buddy-cache.mjs` and
  `skills/openspec-buddy/scripts/ensure-cache-dir.sh` to manage
  `openspec/.buddy-cache/`, install the default `.git/info/exclude` rule, and
  enforce cache identity checks for repo, project, issue, PR, and relationship
  payloads.
- Updated `github-fetch.sh` to reuse cache entries across repo, project, issue,
  PR, ready-scan, and relationship reads; preflight batched GraphQL calls by
  expected call count; and invalidate relationship caches after status and
  relationship mutations.
- Updated `set-project-status.sh`, `set-project-date.sh`,
  `configure-pr-metadata.sh`, and `verify-pr-coordination.sh` to use
  target-scoped cached metadata instead of project-wide item enumeration.
- Updated `request-pr-review.sh` and `mark-review.sh` so review-request flows
  stay on REST payloads and leave review-thread GraphQL checks to
  `wait-for-review-clear.sh` and `verify-review-clear.sh`.
- Expanded eval coverage for cache identity mismatches, project metadata cache
  refresh, GraphQL budget fail-closed behavior, relationship-cache
  invalidation, and status-label cache invalidation.

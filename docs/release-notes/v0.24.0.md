# v0.24.0

## Added

- Added a deterministic `## Testing Strategy` validator for proposal
  `design.md` and `.buddy/issue.md` artifacts.
- Added propose-time testing-strategy validation after issue-body and proposal
  shape gates, before any GitHub Issue mutation.

## Changed

- Behavioral, medium-risk, and high-risk changes now declare observable public
  behavior, the highest public test seam, seam reuse, and per-AC evidence.
- Documentation and mechanical changes may use `not-applicable` with an
  explicit verification method and rationale.
- Manual-only acceptance now records both why automation is not applicable and
  the manual evidence check for each AC.
- Apply and Buddy Auto consume the approved seam without restarting product
  seam selection. Optional Matt TDD changes implementation method only; the
  Buddy-native fallback preserves red-before-green public-interface testing.

## Fixed

- Scoped AC extraction to unchecked items in the unique Acceptance Checklist,
  so historical or external AC references elsewhere do not block proposals.
- Distinguished empty negative placeholders from substantive explanations such
  as `not automated because ...`.

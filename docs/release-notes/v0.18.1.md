# v0.18.1

## Fixed

- Fixed `reply-review-thread.sh` to avoid querying the unsupported
  `comment.pullRequestReviewThread` GraphQL field when adding same-thread
  review replies.
- Hardened post-merge achievement from the bound coordination branch. The
  post-merge path now verifies PR merged state, origin issue, archive path,
  completed archived tasks, and issue `change_id` before issue or Project
  terminal state can be changed.
- Fixed post-merge review thread verification so merged PR closeout can run
  from the bound branch without requiring the old claim branch worktree.
- Fixed achievement truth checks when GitHub REST `projectItems` omits the
  Project `End` field by adding a targeted Project field GraphQL fallback only
  after merge and archive gates pass.
- Fixed series parent closeout so unfinished or drifted sibling issues do not
  block the current child issue closeout. Direct parent reconciliation still
  fails on repairable child terminal drift.
- Made Buddy Auto retry deterministic post-merge achievement once when a helper
  explicitly reports `safe_to_rerun: true`.

## Verification

- `npm test`
- `npm pack --dry-run`
- Independent subagent review of staged post-merge achievement hardening.
